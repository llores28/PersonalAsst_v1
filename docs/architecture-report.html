2. Message Lifecycle — Request Flow

Detailed sequence of what happens when a user sends a message.

sequenceDiagram
    actor User
    participant TG as Telegram Bot
    participant Queue as User Queue
    participant Guard as Safety Agent
    participant Orch as Orchestrator
    participant LLM as OpenAI API
    participant Session as Redis Session
    participant Tools as Skill Tools
    participant MCP as Workspace MCP
    participant Mem as Memory (Mem0)

    User->>TG: Send message
    TG->>TG: Auth check (allowlist)
    TG->>Queue: Enqueue (per-user sequential)
    Queue->>Guard: Input guardrail check
    Guard->>LLM: Injection detection (fast model)
    LLM-->>Guard: safe / blocked
    alt Input blocked
        Guard-->>User: ⚠️ Blocked message
    end
    Guard->>Orch: Prepare input
    Orch->>Orch: Complexity routing (nano/mini/standard/pro)
    Orch->>Session: Load conversation history (last 20 turns)
    Orch->>Orch: Build persona prompt + skill instructions
    Orch->>LLM: Runner.run() with tools + session
    loop Tool calls (max turns)
        LLM-->>Orch: Tool call request
        Orch->>Tools: Execute tool
        Tools->>MCP: Google API call (if workspace tool)
        MCP-->>Tools: Result
        Tools-->>Orch: Tool result
        Orch->>LLM: Continue with result
    end
    LLM-->>Orch: Final response
    Orch->>Guard: Output PII check (context-aware)
    alt PII detected (not email workflow)
        Guard-->>User: ⚠️ Filtered response
    end
    Orch->>Session: Save to SDK RedisSession
    Orch->>Mem: Store interaction (if memorable)
    Orch-->>User: 📨 Response via Telegram

🔐
Authentication
DB-backed Telegram user ID allowlist. Owner manages via /allow and /revoke commands.
🛡️
Input Guardrail
Pattern matching + fast LLM check (gpt-4.1-nano). Blocks prompt injection attempts.
🔍
Context-Aware PII
Two-layer output check: (1) user message context for email workflows, (2) output marker fallback. Prevents false positives.
📦
Tool Sandbox
Generated CLI tools run in empty environment. No access to API keys. Timeout + static analysis + user approval.
💰
Cost Caps
Daily and monthly spend limits. 80% alert, 100% block. Tracked in PostgreSQL.
📋
Audit Logging
Every interaction logged: agent, tools used, cost, duration, errors.

9. Key Design Decisions

Architectural choices that define the system.

DecisionChoiceRationale
Single async processBot + agents + scheduler in one Python processSimplicity. One process, one deployment unit.
Direct tools over agent wrappersAll 45 Workspace tools are function_tool closuresSingle LLM call per request. No nested agent reasoning overhead.
_impl pattern for bound toolsPlain async functions, not FunctionTool objectsFunctionTool is not callable. Prevents runtime TypeError.
SDK RedisSessionReal conversation turns, not system-prompt embeddingLLM sees tool results, resolves references like "send it".
Context-aware PII guardrailsUser message passed via Runner contextPrevents false positives on email workflows.
Defensive MCP integrationStrip None values, correct schema fieldsMCP server uses additionalProperties: false.
Sequential per-user queueasyncio.Queue per user_idPrevents race conditions in multi-turn conversations.
CLI-first tool creationargparse scripts over MCP serversSimpler, sandboxable, no discovery overhead.
Prompt cache optimizationStatic content first, dynamic content last41-80% cost reduction via OpenAI prompt caching.
Memory deduplication0.85 cosine similarity check before storePrevents duplicate memories accumulating.
Stale session recoveryCatch BadRequestError, clear session, retry onceGraceful recovery from corrupt SDK sessions.
Startup migration safety gate`STARTUP_MIGRATIONS_ENABLED` defaults to falsePrevents unsafe startup migration execution by default; explicit ops control.
Dashboard org ownership enforcementResolve requester (`X-Telegram-Id`) + owner-scoped org accessPrevents cross-org mutation and unauthorized dashboard operations.
Durable org delete auditWrite `audit_log` entry before cascading org deletePreserves deletion evidence even when org activity rows are removed by cascade.
CORS allowlist hardening`CORS_ALLOWED_ORIGINS` parser removes wildcard and deduplicatesTightens API exposure to explicit trusted dashboard origins.
One-shot scheduler sync normalizationPersisted `run_at` normalized to canonical ISO before schedulingEliminates type/contract mismatch between DB sync and scheduler engine.
Repair agent honestyNo codebase access, cannot fabricate pathsPrevents hallucinated patches that confuse users.
Repair risk classification (M4)classify_repair_risk() → low/medium/high; low auto-appliedLets routine operational fixes happen without friction while keeping code changes gated.
Post-apply smoke test (M4)verify_repair() runs pytest; rollback_repair() on failurePrevents auto-applied fixes from leaving system in broken state.
Parallel fan-out (M1)asyncio.gather, max 3 branches, 80% budget guardMulti-domain requests complete simultaneously; budget guard prevents runaway cost.
Background job tick loop (M2)APScheduler interval + fault counter + iteration capAutonomous jobs self-terminate safely on success, fault overflow, or cap reached.
Agent trace persistence (M3)Every FunctionCallItem persisted to agent_traces after Runner.run()Full step-by-step observability without LLM overhead; queryable via API.
Generic web scrapercrawl4ai + SiteLoginConfig registryZero per-site scraper code. Add a site by declaring login selectors. LLM parses markdown itself.
Tools consolidated into src/tools/ merged into src/tools/plugins/Single source tree. Linting, typing, imports consistent. Tool Factory writes to same tree.
Persona interview (Stanford)3-session structured OCEAN profilingReplicates personality 85% accurately per Stanford research. Deep profile injected into system prompt.
PruningContentFilter for SPAscrawl4ai threshold=0.48 + _clean_markdown()LinkedIn pages have 400K+ of JSON blobs. Filter strips noise, post-processor removes tracking data.

Key reliability/security bug fixes and hardening milestones (March-April 2026).

Bugs Fixed by Category

Owner maintenance requests flagged as unsafe. Fix: _is_owner_maintenance_request() bypass.

Calendar returned wrong dates (June not March)
Persona prompt had no current datetime. Model guessed wrong. Fix: runtime datetime injection.
March 19, 2026
PII guardrail blocked email drafts
Narrow output marker allowlist. Fix: expanded markers for draft/send phrasings.
March 19, 2026
"Something went wrong" for output guardrail trips

"Send it" after draft loops forever
Pending payload gate too narrow. Fix: broader _is_email_related_request() check.
March 19, 2026
Tasks routed to calendar agent
No task-capable tool exposed. Fix: explicit tool routing rules + scheduler binding.
March 20, 2026
Gmail send rejected by MCP server
reply_to_message_id not in MCP schema. Fix: replaced with thread_id/in_reply_to.
March 20, 2026
Stale session 400 errors
Orphaned function_call_output items in SDK session. Fix: filter history + catch-and-retry.
March 20, 2026
Reminder tool errored — DateTrigger API
APScheduler 4.x uses run_time, not run_date. Fix: parameter rename + timezone hardening.
March 20, 2026
FunctionTool object is not callable
Bound tools called @function_tool objects directly. Fix: extracted _*_impl pattern.
March 20, 2026
Repair agent hallucinated patches
No codebase access but fabricated file paths. Fix: honest instructions about limitations.
March 21, 2026
LinkedIn Voyager profileView API deprecated (410)

crawl4ai returned JSON blobs instead of rendered content

March 21, 2026
crawl4ai login hook timed out on LinkedIn
wait_until="networkidle" stalls on JS-heavy SPAs. Fix: switched to "domcontentloaded", increased timeout to 60s.
March 22, 2026
Dynamic tools invisible after consolidation

April 10, 2026
Dashboard org mutations lacked owner scoping
Org endpoints accepted broad access. Fix: requester resolution and owned-org checks across dashboard org API mutations.
April 10, 2026
Org deletion had non-durable audit trail
Org activity entries could cascade-delete with org data. Fix: durable `audit_log` write before delete (API + Telegram `/orgs delete`).
